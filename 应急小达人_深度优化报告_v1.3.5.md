## 二、性能与加载：资源打包 + 压缩

**问题**：原站以 47 个独立 `<script>`（共 1.56 MB JS）与 13 个 `<link>`（共 454 KB CSS）加载，带来 66 个 HTTP 请求与重复的解析/编译开销。

**方案**：
- 按 `index.html` 原有加载顺序，将 45 个 `defer` 脚本拼接为单个 `js/app.min.js`（**严格保留全局作用域与执行顺序**，与原"多个独立 script 共享全局"语义完全一致，零语义风险）。
- 将 9 个非关键样式表按原顺序合并压缩为 `css/all.min.css`；保留 `critical.css` 单独加载以保障首屏（FCP）。
- JS 用 terser **仅做安全压缩（不重命名全局）**，避免破坏 HTML 内联脚本对 `PageManager.navigate` 等全局函数的调用；CSS 用 csso（正确处理 `calc()` / `url()`）。
- 配套更新 `tools/build-bundle.cjs` 可复现构建脚本（固定有序源清单，避免自引用）。

**结果**：请求数 66 → 14；JS 原始 931 KB → 780 KB（gzip 282 KB）；CSS 原始 233 KB → 170 KB（gzip 29 KB）。

## 三、运行时与渲染：降低主线程与绘制成本

1. **跳过非激活页面的布局/绘制**（`scroll-fix.css`，沿用并随本次提交）
   为 `.page:not(.active)` 添加 `content-visibility:auto` + `contain-intrinsic-size`，浏览器跳过 50+ 个满屏隐藏页面的布局与绘制；激活页加 `contain: layout style` 做渲染隔离，显著降低首屏布局成本与页面切换抖动。

2. **高级粒子背景减负**（`bg-premium.js`）
   - 尊重系统 **"减少动态效果"** 设置：开启时仅绘制一帧静态背景，不再进入 `requestAnimationFrame` 持续循环。
   - 监听 `visibilitychange`：标签页隐藏时暂停动画、回到前台恢复，降低后台 CPU/电量开销。

3. **灾害粒子引擎**：经核查已为懒启动（仅 hook 到 `DisasterSimEngine.animate`，在灾害模拟页才运行），无需改动，无加载期开销。

## 四、无障碍（A11y）

新增 `a11y-polish.css`：
- **`prefers-reduced-motion`**：系统开启"减少动态效果"时，全局停用非必要动画/过渡（动画时长归零、滚动行为改为 auto），兼顾前庭功能敏感用户与低性能设备。
- **`:focus-visible` 焦点环**：键盘导航时显示清晰的蓝色焦点环（3px + 外发光），鼠标点击不产生干扰性描边，提升键盘可达性。
- 设置页 **AI 模型选择器**样式适配深色主题。

---

## 五、AI 导师：DeepSeek 推理模型可切换

针对"DeepSeek 调用哪个模型"的疑问，在**设置 → AI 导师**新增"推理模型"选择器：
- 可选 **V3 标准（deepseek-chat，快速）** 与 **R1 推理（deepseek-reasoner，更强逻辑）**。
- 选择即写入 `localStorage['aitutor_model']` 并调用 `DeepSeekAPI.setModel()`，下次对话即时生效；进入设置页时自动回填已保存值。
- 实测：`deepseek-chat → deepseek-reasoner` 后，`localStorage` 与 `DeepSeekAPI._model` 同步更新、描述文案正确。

## 七、安全与兼容说明

- **保留旧的 45 个独立 JS 文件**不删除：仅停止在 `index.html` 中引用。原因——已安装旧 Service Worker 的用户可能缓存了旧版 HTML 并引用旧脚本，保留可避免 404；新 SW（`CACHE_NAME` 升级至 `yingji-xiaodaren-v90`）会预缓存新 bundle 并自动接管。
- 打包严格按原顺序拼接、零重命名，确保与原有全局作用域行为完全一致；已通过 51 页遍历 + 0 错误回归验证。
